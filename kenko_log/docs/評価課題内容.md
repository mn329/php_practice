# Kenko Log 未完成箇所調査レポート課題

## 課題の目的

「Kenko Log」の未完成箇所や動作不良を自分で調査し、原因と修正方針をレポートとしてまとめる。

- エラーや表示崩れを見つける力
- PHP、JavaScript、SQL、HTML フォームの役割を説明する力
- データベース定義や既存コードを根拠に原因を考える力
- 修正後に何を確認すればよいかを整理する力
- マークダウン（md）の記述訓練

## 作業ルール

- 「kenko_log」プロジェクトを、WebサーバとMySQLで動作させる。「
- DBを作成するとテストユーザでログインできる。※ログイン画面で「テストユーザーを入力」でログイン可能
- 仕様は `docs/md/database.md`、`docs/md/web_interfaces.md` を参照する。
- プロジェクト内の PHP / HTML / JS を調査する。
- レポートには「どのファイルを見たか」「問題点・修正点」を記述する。
- AI を使った場合は、どのように使ったかを最後に明記する。

## 必須条件

- `docs/未完成箇所レポート.md` にある項目から、**最低 8項目を調査**する。
- 「未完成・要修正箇所」一覧を参考に、**症状・原因・修正内容・動作確認** をまとめる。

## レポート項目

| 観点 | 内容 |
|---|---|
| 症状の確認 | 実際の画面、エラー、表示内容を説明 |
| 原因の説明 | ファイル名、変数名、SQL、API URL などを根拠に説明 |
| 修正方針 | 既存の設計や DB 定義に合った修正方針 |
| 動作確認 | 修正後に確認すべき操作と期待結果 |

例）

```md
### No.5 健康記録の追加

#### 症状
健康記録を追加すると SQL エラーが表示される。

#### 確認したファイル
health/insert.php

#### 原因
SQLがコーディングされていない。

#### 修正内容
SQLを以下のように修正した。

INSERT INTO ....

#### 動作確認
```
